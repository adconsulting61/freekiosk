1 Deploy this server (if not already done)

Deploy on Railway — Free 500 hrs/month

  1. Create a free account at railway.app
  2. Create a new GitHub repository and push the relay-server/ folder into it
  3. In Railway: click New Project → Deploy from GitHub repo
  4. Select your repo. Railway auto-detects Node.js and deploys.
  5. Go to your project → Variables → add STREAM_SECRET = any random string you choose
  6. Go to Settings → Networking → Generate Domain — copy the URL (e.g. freekiosk-relay.up.railway.app)
  7. Your WebSocket URL: wss://freekiosk-relay.up.railway.app
When Railway free hours run out, stop the Railway project and switch to the Windows PC option below. Your settings stay saved in the kiosk app — just update the relay URL.

Deploy on Render — Free Tier

  1. Create a free account at render.com
  2. Push the relay-server/ folder to a GitHub repo
  3. In Render: New → Web Service → Connect GitHub repo
  4. Build command: npm install   Start command: node server.js
  5. Add environment variable: STREAM_SECRET = your chosen secret
  6. Click Deploy. Your URL will be wss://your-app.onrender.com
Render free tier spins down after 15 min of inactivity. First connection after sleep takes ~30 seconds.

Run on Windows PC (always-on)

  1. Copy the relay-server folder to your PC
  2. Right-click install-windows.ps1 → Run with PowerShell as Administrator
  3. The installer will: install Node.js, set up the server, open firewall port 3000, and start it automatically on reboot
  4. Note the secret key printed at the end
  5. For internet access: log into your router and set up port forwarding: external port 3000 → your PC's local IP on port 3000
  6. Find your public IP at whatismyip.com
  7. Relay URL: ws://YOUR-PUBLIC-IP:3000
For a stable URL even if your home IP changes, use a free dynamic DNS service like DuckDNS. Then your URL stays the same like ws://yourname.duckdns.org:3000

Run on Linux VPS or Mac

# Copy relay-server folder to your server, then:
bash install-linux.sh

The script installs Node.js, sets up the server as a background service with pm2, and opens port 3000.

# Check status
pm2 status

# View live logs
pm2 logs freekiosk-relay

2 Configure the kiosk app

In the FreeKiosk Android app:

  1. Enter your admin PIN to open Settings
  2. Go to the Advanced tab
  3. Scroll to Remote Monitoring
  4. Toggle Enable Remote Screen Monitoring ON
  5. Enter your Relay Server URL (the ws:// or wss:// address from Step 1)
  6. Tap Start Streaming
  7. Android will show a "Start recording?" dialog — tap Start now
  8. The tablet is now streaming
The screen capture consent dialog appears once per session. After a reboot, you need to tap "Start Streaming" again (or it can auto-start — coming in a future update).

3 Watch from the admin dashboard

Open the dashboard

  1. Open the dashboard in any browser
  2. Enter your relay server URL in the bottom bar (pre-filled if you're already on the server)
  3. Enter your secret key
  4. Click Save
  5. Your tablet appears in the left sidebar — click it to start watching
  6. Use the 📷 Snapshot button to save a screenshot
Stream runs at ~2 frames per second by default — enough for monitoring. The frame rate is configurable in the kiosk app settings.

4 Switching from Railway to your PC mid-month

  1. Run install-windows.ps1 on your PC (Step 1, Windows tab)
  2. Get your new relay URL (ws://YOUR-PUBLIC-IP:3000)
  3. In the kiosk app: Settings → Advanced → Remote Monitoring → update the URL
  4. Tap Stop Streaming, then Start Streaming again
  5. In Railway: pause or delete the project to stop counting hours
The kiosk's device ID stays the same — your admin dashboard will reconnect automatically once you update the relay URL.
